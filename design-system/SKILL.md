Read the `readme.md` file within this skill, and explore the other available files
(`styles.css` + `tokens/` for the foundations, `components/` for the primitives,
`ui_kits/github-battle/` for full-screen recreations).

If creating visual artifacts (slides, mocks, throwaway prototypes), copy assets out
and create static HTML files that link `styles.css` for the tokens. If working on
production code (Next.js 16 · Tailwind 4 · shadcn/ui "new-york"), lift the token
values and component patterns directly.

If the user invokes this skill without other guidance, ask them what they want to
build, ask a few focused questions, and act as an expert designer for the GitHub
Battle brand — outputting HTML artifacts or production code as the need dictates.
Keep it calm, warm-neutral, serif-led numbers, purposeful motion. Never add emoji
or icon slop; the product is intentionally near-iconless.
